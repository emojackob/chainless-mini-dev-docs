#### 1. 用戶訪問

- 無鏈錢包用戶通過 無鏈錢包 App 或 小程序 WebView 訪問小程序。

- 訪問過程中調用官方小程序服務器、第三方小程序前端（如 PUMP、SWAP 等）和業務後端。

#### 2. 小程序開發與審核

- 第三方開發者需先在官方平台註冊開發者賬號。

- 開發者提交小程序的前端頁面與智能合約至 官方 GitHub 審核倉庫，包含配置徽章、圖標、小程序入口等資訊。

#### 3. 小程序架構與部署

- 小程序分為不同類型（如 PUMP、SWAP、理財產品等），由開發者自行搭建：

- 小程序前端通過 API 調用自建的小程序業務後端與資料庫。

- 開發者可以獨立搭建自己的服務，不受平台限制。

#### 4. 合約發布與區塊鏈交互

- 小程序運營方（開發者）將合約與頁面一併發布。

- 用戶通過官方提供的 API/SDK 實現：身份驗證、建立鏈接、發起交易

- 官方 API 將與官方數據庫和無鏈主鏈（Blockchain）進行互動，完成資料查詢、交易簽名、鏈上提交等操作。
